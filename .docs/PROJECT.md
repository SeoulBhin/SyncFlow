# SyncFlow — 프로젝트 개요

> **AI가 회의에 참여하는 스마트 협업 플랫폼**

## 경쟁 포지셔닝 — vs. Jira 스택

### Jira의 구조적 약점

Jira는 업계 표준 프로젝트 관리 도구이지만, **단독으로 협업이 불가능**하다:

| 기능 | Jira 단독 | 필요한 추가 도구 | 추가 비용 |
|------|----------|----------------|----------|
| 프로젝트 관리 | ✅ | - | - |
| 문서 협업 | ❌ 동시 편집 불가 | Confluence | +$5~10/user/월 |
| 팀 채팅 | ❌ 없음 | Slack | 별도 구독 |
| 음성/화상 회의 | ❌ 없음 | Zoom | 별도 구독 |
| AI 어시스턴트 | ⚠️ Premium 전용 | - | $14.54+/user/월 |
| **합계** | | **4~6개 도구** | **$25~40+/user/월** |

### SyncFlow vs Jira 스택

| 기능 | Jira 스택 | SyncFlow |
|------|----------|----------|
| 프로젝트/작업 관리 | ✅ Jira | ✅ 칸반/캘린더/간트/리스트 + 커스텀 상태 |
| 문서 협업 | ⚠️ Confluence (동시 편집 제한) | ✅ CRDT 실시간 동시 편집 |
| 팀 채팅 | ❌ Slack 필요 | ✅ 내장 메신저 + 스레드 |
| 음성 회의 | ❌ Zoom 필요 | ✅ LiveKit 내장 음성 |
| **AI 회의록 자동 생성** | ❌ 없음 | ✅ STT + RAG 기반 |
| **회의→작업 자동 등록** | ❌ 수동 Jira 티켓 | ✅ AI 자동 추출 → 리뷰 → 일괄 등록 |
| **프로젝트 맥락 AI** | ⚠️ Premium, JQL 수준 | ✅ RAG 전체 문서/코드/회의록 검색 |
| 코드 실행 | ❌ 없음 | ✅ Docker 샌드박스 (부가) |
| **도구 개수** | **4~6개** | **1개** |
| **비용** | **$25~40+/user/월** | **단일 구독** |

### SyncFlow만의 고유 시나리오

| 시나리오 | Jira 스택 | SyncFlow |
|---------|----------|----------|
| 회의 중 "저번에 어떻게 결정했지?" | Confluence 검색 → 못 찾음 | AI가 이전 회의록에서 즉시 검색 |
| 회의 끝 → 작업 등록 | 수동 회의록 + 수동 Jira 티켓 | AI 회의록 자동 + 액션아이템 리뷰 → 일괄 등록 |
| "이 문서 같이 수정하자" | Confluence 동시 편집 제한 → 충돌 | CRDT 실시간 동시 편집 + 음성 대화 |
| 기술 문서 코드 시연 | "직접 돌려보세요" | 문서 안 코드 클릭 → 바로 실행 |

## 핵심 기능 (3+1)

### ① AI 회의 어시스턴트 — 킬러 피처

LiveKit 음성 + STT + RAG AI를 결합하여 **AI가 회의에 참여하는 경험** 제공.

- **실시간 STT**: 회의 중 대화를 실시간 텍스트 변환, 화자 분리 포함
- **AI 회의록 자동 생성**: 프로젝트 맥락(RAG)을 이해한 구조화된 요약
- **액션아이템 리뷰 → 작업 일괄 등록**: AI가 추출한 작업을 사용자가 편집/확인 후 작업보드에 일괄 등록. "회의에서 생성" 배지 부착
- **관련 자료 자동 제안**: 회의 중 프로젝트 문서·이전 회의록을 AI가 검색하여 제안

> Jira+Zoom에 없는 기능. 클로바노트 등 STT 도구는 프로젝트 맥락을 모르고 작업보드 연동도 안 됨. "회의→회의록→작업 등록"이 자동으로 한 플랫폼에서 이루어지는 건 현재 없음.

### ② RAG 기반 프로젝트 컨텍스트 AI

Google Gemini API + pgvector RAG 파이프라인으로 **프로젝트 전체를 이해하는 AI**.

- 문서·코드·회의록 자동 청킹 → 768차원 벡터 임베딩 → pgvector 저장
- 코사인 유사도 검색으로 관련 문맥 추출 → AI 프롬프트에 주입
- "지난 회의에서 결제 건 어떻게 결정했지?" → 해당 회의록을 찾아 답변

### ③ CRDT 기반 실시간 문서 동시 편집

Yjs CRDT + Socket.IO + TipTap/Monaco로 **구글 독스 수준의 실시간 동시 편집**.

- 여러 사용자가 같은 문서를 동시에 편집, 밀리초 단위 반영
- 라이브 커서 (이름·색상 구분)
- 문서(TipTap) · 코드(Monaco) · 작업보드 3종 동시 동기화

### ④ Docker 코드 실행 (부가 기능, 후순위)

- 문서 안에서 코드를 바로 실행 — 시연, 검증, 간단한 테스트 용도
- 7개 언어 (Python, JS, Java, C, C++, HTML, CSS), Docker 보안 샌드박스
- 풀 IDE가 아닌 "실행 가능한 코드 블록" 포지셔닝
- ⏸️ 핵심 기능(AI 회의, RAG, 실시간 편집) 구현 완료 후 진행 예정

## 설계 결정 사항

| 항목 | 결정 | 근거 |
|------|------|------|
| **네비게이션** | Slack 스타일 통합 사이드바, 하단 툴바는 액션만 | 채널/DM/프로젝트를 접을 수 있는 섹션으로 통합. 하단에서 채널 피커 제거 |
| **스레딩** | Slack 스타일 단일 레벨 스레드 | 메인 채널 + 오른쪽 스레드 패널 분리. "N개 답글" 인디케이터 |
| **작업 관리** | 커스텀 상태(5-7컬럼) + 서브태스크 + 고급 필터 | 채널별 설정 가능한 상태 컬럼. 서브태스크 체크리스트. 담당자/우선순위/날짜/키워드 필터 |
| **회의 파이프라인** | 액션아이템 리뷰 스크린 추가 | 회의 종료 → AI 추출 → 사용자 편집/확인 → "작업보드에 등록" 일괄 버튼 |
| **작업-소통 연동** | 양방향 연결 + 자동 활동피드 | 채팅에서 #작업명 임베드, 메시지→작업 생성, 상태 변경 시 채널 자동 알림 |
| **권한 모델** | 4역할 + 채널별 오버라이드 | Owner/Admin/Member/Guest + channel_members.permission_override JSONB |
| **Docker** | 부가 기능으로 포지셔닝 | 킬러 피처가 아닌 차별화 보조. 라이브러리 설치 문제는 기본 이미지에 사전 설치로 완화 |

## 핵심 기술 — 라이브러리 vs 자체 설계

| 핵심 기술 | 라이브러리가 해주는 것 | 우리가 직접 설계·구현하는 것 |
|----------|---------------------|--------------------------|
| **AI 회의 어시스턴트** | STT API가 음성→텍스트 변환 | STT 파이프라인, 화자 분리, 회의 분석, 작업 추출, 리뷰 플로우, RAG 연동 |
| **RAG 파이프라인** | Gemini API 텍스트 생성, pgvector 벡터 저장 | 청킹 전략, 임베딩 갱신, 검색 튜닝, 프롬프트 엔지니어링 |
| **CRDT 동기화** | Yjs CRDT 자료구조, Socket.IO WebSocket | 동기화 서버 커스텀, 다중 문서 타입 동기화, JWT 인증·권한 연동 |
| **Docker 샌드박스** | Dockerode 컨테이너 API | 보안 정책, 7개 언어 환경, 입출력 스트리밍 |
