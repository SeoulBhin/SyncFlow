# SyncFlow → Slack 스타일 UI/UX 개선 제안서

> 작성일: 2026-05-14
> **전제**: 기능·API·DB 스키마는 일절 건드리지 않는다. 순수 UI/UX (레이아웃, 컴포넌트 배치, 색 시스템, 인터랙션 패턴) 변경만 제안한다.
> **목표**: "Slack을 따라하고 싶다" — 사용자 익숙함을 최대로 활용하고, 인지 부담을 줄인다.

## 현재 UI 구조 (요약)

```
┌─────────────────────────────────────────────────────────────────┐
│  SlackHeader (h-12, 좌: 로고 / 중: 검색 / 우: AI·테마·유저메뉴)  │
├──────────────────────────────────────────────────────────────────┤
│  MeetingBanner (조건부)                                          │
├──────────────────────────────────────────────────────────────────┤
│  AIContextBanner (※ 하드코딩 데모 메시지 — PROJECT_ISSUES #4)    │
├────────────┬─────────────────────────────────┬──────────────────┤
│            │                                  │                  │
│  Slack-    │     <main> Outlet                │  DetailPanel     │
│  Sidebar   │     (Channel / Tasks / Meetings) │  (AI/Voice/Thread)│
│  260px     │                                  │  380px           │
│            │                                  │                  │
└────────────┴─────────────────────────────────┴──────────────────┘
```

### 현재 좋은 점
- 사이드바 컨셉은 Slack에 가깝다 (조직 헤더 + 즐겨찾기/채널/프로젝트/DM 섹션 + 프로필·설정)
- 우측 DetailPanel 슬롯 패턴이 잘 분리되어 있음 (AI/음성/스레드 토글)
- 그룹 컨텍스트 switching이 Discord/Slack 패턴과 유사

### Slack과 가장 큰 차이 (= 개선 여지)
1. **워크스페이스 스위처가 사이드바 헤더에 통합** — Slack은 사이드바 **왼쪽 옆에 별도 레일(rail)**로 워크스페이스 아이콘을 세로로 쌓는다. 현재는 dropdown 하나로만 전환.
2. **상단 글로벌 헤더가 두껍다 (h-12)** — Slack은 상단 헤더가 더 슬림하거나(28-36px) 아예 사이드바 안으로 흡수돼 있음.
3. **검색바가 정중앙 고정** — Slack 최신 UI는 검색이 헤더 정중앙에서 더 넓게 차지하며 페이지 컨텍스트 자체.
4. **채널 헤더에 액션 슬롯이 비어 보임** — Slack은 채널 헤더 우측에 멤버/핀/파일/통화 시작 등 빠른 진입점이 늘어서 있음.
5. **DM/채널 분리가 명확하지 않음** — Slack은 즐겨찾기/채널/DM/앱 등이 시각적으로 더 강하게 그루핑.
6. **메시지 hover 액션** — Slack은 메시지 위에 둥둥 떠있는 reaction/reply/save/more 4개가 표준. 현재는 위치/일관성 점검 필요.
7. **AI 진입점이 우상단 별 아이콘 1개** — Slack은 Slack AI를 검색바 내부 + 사이드바 상단 양쪽에 노출.

## 제안 A: 워크스페이스 스위처 사이드바 (Slack의 좌측 레일)

### 무엇을 바꾸나
사이드바 왼쪽에 **w-16 (64px) 폭의 세로 레일**을 추가한다. 사용자가 속한 조직을 아이콘으로 쌓아 보여주고, 클릭 시 즉시 전환. 현재의 dropdown 방식은 그대로 유지 (보조 진입점).

```
[조직레일][SlackSidebar  ][      메인       ][DetailPanel]
   64px     260px → 224px
```

### 영향
- ✅ **이점**: 조직이 3개 이상인 사용자가 한 번에 모든 조직을 인지. Slack/Discord 사용자 학습 비용 0.
- ✅ 활성 조직 표시(파란 막대)로 어디 있는지 즉시 인지.
- ✅ 알림 뱃지를 조직 단위로 표시 가능 → 다른 조직에 새 메시지 왔다는 신호.
- ⚠️ **단점**: 가용 너비 64px 손실. 1080p에서 콘텐츠가 살짝 좁아짐.
- ⚠️ 조직이 1개만 있는 학생/소규모 사용자에게는 잉여. → 조직 1개일 때는 자동 숨김.
- 변경 범위: `AppLayout.tsx`에 `OrgRail` 컴포넌트 1개 추가, `SlackSidebar` 폭만 축소. **API 변경 0**.

### 채택 시 변하는 것
- `useGroupContextStore`는 이미 `myGroups[]`를 갖고 있으므로 데이터는 그대로
- `SlackSidebar`의 헤더 dropdown은 유지 (검색·관리용)
- mobile은 햄버거 메뉴 안에 흡수

## 제안 B: 글로벌 헤더 축소 + 검색바 확장

### 무엇을 바꾸나
- 현재 `SlackHeader` 높이 h-12 (48px) → **h-9 (36px)** 로 축소
- 로고는 사이드바 안 조직 헤더 위로 이동 (또는 조직 레일 최상단)
- 헤더 가운데 검색바를 **`max-w-2xl`로 확장** + placeholder를 `"채널, 메시지, 사람, 파일 검색"`처럼 명시
- 우측 아이콘은 AI / 알림 / 도움말 / 유저메뉴 4개로 정렬

### 영향
- ✅ 콘텐츠 영역 세로 12px 회수 → 메시지/태스크 더 보임
- ✅ 검색이 시각적으로 더 중요해짐 — Slack은 검색이 최상위 입력
- ✅ AI 별 아이콘을 그대로 두면 진입점 손실 없음
- ⚠️ 로고가 사이드바로 들어가면 비로그인 페이지 헤더와 디자인 분리됨 (PublicLayout은 그대로)
- ⚠️ 우측에 알림 종/벨 아이콘을 새로 추가하려면 알림 inbox API가 필요 — **이번 제안에선 placeholder UI만** (Phase 2 백엔드 작업과 맞물림)
- 변경 범위: `SlackHeader.tsx`만. **API 변경 0**.

## 제안 C: 채널 헤더에 빠른 액션 슬롯 추가

### 무엇을 바꾸나
현재 채널 헤더에는 "# 채널이름 / 토픽 / ⚙️" 정도가 있다. Slack 스타일로:
```
# 채널명  토픽▾    │  👥 멤버 12  📌 핀  📁 파일  📞 통화 시작  ⚙️
```

- 멤버: 클릭 시 DetailPanel members 슬롯 열림 (현재 "준비 중" 상태 → 디자인만 먼저)
- 핀: 클릭 시 핀된 메시지 목록 (Phase 1)
- 파일: 채널의 파일 목록 (Phase 2)
- 통화 시작: 현재 회의 시작 버튼과 동일 (단순 위치 이동)
- ⚙️: 현재 위치 유지

### 영향
- ✅ Slack 사용자에게 0학습 곡선
- ✅ DetailPanel slots(`members`/`pinned`/`files`)을 향한 자연스러운 진입점 — 백엔드 API가 늦게 따라와도 UI는 placeholder로 미리 노출 가능
- ⚠️ 핀/파일은 아직 백엔드 API 없음 → 클릭 시 "곧 출시" 토스트 또는 비활성 처리. 사용자가 누르려다 못 누르면 불만 가능 → **빈 상태에서는 아예 안 보이게 토글**도 가능.
- 변경 범위: `ChannelHeader.tsx`만. **API 변경 0** (멤버 카운트는 이미 `useChannelsStore`로 들고 있음).

## 제안 D: 사이드바 섹션 간격·계층 강화

### 무엇을 바꾸나
- 섹션 헤더(즐겨찾기/채널/프로젝트/DM)의 폰트 사이즈 10px → **11px**, 자간 살짝 줄이고 위 padding 6px → 10px
- 채널 아이템 polish: 활성 채널의 좌측에 **2px 파란 막대** 추가 (현재는 배경 색만)
- 안 읽은 채널은 **굵게(font-medium)** 처리, 읽은 채널은 회색 (Slack 핵심 UX)
- 채널/DM/프로젝트 사이를 구분선이 아닌 **공백 8px**로 — Slack은 sub-section 사이에 hairline 거의 안 씀

### 영향
- ✅ "어디에 새 메시지가 있는지" 1초 안에 파악 (현재는 빨간 뱃지만 봐야 함)
- ✅ 활성 채널 인지 속도 향상
- ⚠️ "안 읽음" 굵게 처리는 백엔드 unread 카운트가 정확해야 의미가 있음 — 현재 `unread`는 mock 기반이라 우선 활성/비활성 굵기만 적용. unread 굵기는 Phase 1 백엔드와 함께.
- 변경 범위: `SlackSidebar.tsx` + 자식 컴포넌트 Tailwind 클래스만. **API 변경 0**.

## 제안 E: 메시지 hover 액션바 통일

### 무엇을 바꾸나
모든 메시지(채널/스레드/DM 공통)에 hover 시 **우상단에 떠있는 액션바** 노출:
```
[😀 반응] [💬 스레드] [🔖 저장] [✏️ 편집(본인만)] [⋯ 더보기]
```
- 위치: 메시지 우상단 코너, `position: absolute`, hover 시 페이드인
- 본인 메시지/타인 메시지 동일 위치 (편집·삭제만 본인 한정)
- 더보기 메뉴: 복사, 링크 복사, 신고 등

### 영향
- ✅ Slack/Discord/Teams 표준 — 학습 비용 0
- ✅ 시각적 일관성 (현재는 reaction이 하단/우측 분산)
- ⚠️ 저장 기능은 Phase 1 백엔드 — 우선 UI는 만들고 클릭 시 토스트 "곧 출시" 또는 hover 비활성
- 변경 범위: `ChannelView.tsx` 메시지 렌더 부분 + 스레드 패널. **API 변경 0** (편집/삭제/반응은 이미 있음).

## 제안 F: AI를 더 자연스럽게 — 사이드바 상단 진입점

### 무엇을 바꾸나
현재 AI는 헤더 우상단 `Sparkles` 1개. Slack AI는 (1) 검색바에서 자연어 질의, (2) 사이드바 최상단 "AI 어시스턴트" 고정 메뉴 두 곳에 노출.

- 사이드바 네비 섹션(`Home/Messages/Tasks/Meetings`) **맨 위에 ✨ AI 어시스턴트** 추가
- 클릭 시 DetailPanel AI 열림 (현재 동작과 동일)
- 메모리에 저장된 사용자 피드백 "자동동작 최소화" → **자동 펼침은 안 함**. 명시적 클릭에서만 열림.

### 영향
- ✅ AI 진입점 + 사용자 멘탈모델 향상 ("얘는 항상 옆에 있구나")
- ✅ 현재 헤더 별 아이콘을 그대로 두고 추가만 하면 됨 — 진입 경로 2개
- ⚠️ AI 모듈이 부분 구현 상태이므로 (PROJECT_ISSUES #2) 키 없는 환경에서 클릭 시 graceful한 에러가 필요
- 변경 범위: `SlackSidebar.tsx` NAV_ITEMS 배열 1줄 추가. **API 변경 0**.

## 제안 G: 빈 상태 (Empty States) 강화

### 무엇을 바꾸나
Slack은 빈 채널/빈 워크스페이스에서 illustration + 명확한 다음 행동 버튼을 준다. 현재 `EmptyState`/`WelcomeOnboarding`이 있지만 활용도가 낮다.

- 빈 채널 진입 시: "이 채널에서 첫 메시지를 보내보세요" + 입력창에 포커스
- 빈 작업 보드: "첫 작업 추가하기" 큰 버튼 + 회의 자동 등록 안내 ("회의 후 액션아이템이 여기로 들어와요")
- 빈 대시보드: 현재 `WelcomeOnboarding` 유지 + 최근 활동 placeholder 카드 3개

### 영향
- ✅ 신규 사용자 첫 5분의 이탈률 감소 (졸프 데모 시연 효과)
- ⚠️ 일러스트 자산 필요 — lucide-react 아이콘으로 대체 가능
- 변경 범위: 페이지별 상단 분기. **API 변경 0**.

## 변경 시 사용자에게 보이는 차이 요약

### Before
- 상단 헤더가 두껍고 검색바가 작다
- 조직 전환은 드롭다운 1개 안에 갇혀 있다
- 채널 헤더는 토픽과 ⚙️만 보인다
- 모든 메시지 hover 액션 위치가 미세하게 다르다
- AI는 우상단 별 아이콘에서만 열린다

### After
- 좌측 조직 레일 → 워크스페이스 즉시 인지
- 슬림 헤더 + 와이드 검색바 → 검색이 1순위 액션이 된다
- 채널 헤더에 멤버·핀·파일·통화 버튼이 정렬 → 한 번에 진입
- 메시지 hover 액션바가 모든 곳에서 동일
- AI 진입점 2개 (헤더 + 사이드바) → 어디서든 한 번에

## 추천 진행 방식

**팀장 결정 사항** (이 문서의 핵심):

1. **전부 한 번에 진행** — 약 11시간 짜리 PR 하나로 묶어 처리. 메모리에 저장된 "Phase 4 완료" 다음 단계로 자연스럽게 연결.
